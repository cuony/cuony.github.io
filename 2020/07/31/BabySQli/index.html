BabySQli

进入是个登录框，发现username里可以注入，然后会说wrong user，换成admin说wrong password，估计是要注出密码登陆，但发现()都被过滤了，估计不是让注出密码，估计要啥骚操作，看源码，有个base32，之后再base64解出

select * from user where username = '$name'

虽然过滤了or，但大写绕过可以联合查询出是三列。

去看师傅们，学到了一个骚操作

在联合查询并不存在的数据时，联合查询就会构造一个虚拟的数据。

这里因为前面知道有三列，盲猜id，username，password。

所以我们联合查询

name=1' union select 1,'admin','65f034c0f853471ed478ceb34164523b'%23&pw=ico

因为这个密码并不存在，但他会构造出这个不存在的密码，这个65f034c0f853471ed478ceb34164523b就是ico的md5，比照就为真，得到flag。
